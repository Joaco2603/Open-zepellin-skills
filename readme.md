

**Summary of Skills**

- **Description:** A collection of notes and guides on patterns, security, and best practices using OpenZeppelin and related tools.
- **Purpose:** Serve as a quick reference for developing, deploying, auditing, and testing secure and efficient smart contracts.

**Skills (files)**

- **OpenZeppelin Overview:** [github/open-zeppelin.md](github/open-zeppelin.md) — An overview of the library and its components.
- **Access Control:** [github/oz-access-control.md](github/oz-access-control.md) — Access control models, roles, and best practices.
- **DeFi Patterns:** [github/oz-defi-patterns.md](github/oz-defi-patterns.md) — Common patterns in DeFi protocols.
- **Deployment Scripts:** [github/oz-deployment-scripts.md](github/oz-deployment-scripts.md) — Scripts and recommendations for automated deployments.
- **ERC20:** [github/oz-erc20.md](github/oz-erc20.md) — Implementation details and nuances for fungible tokens.
- **ERC721 / ERC1155:** [github/oz-erc721-erc1155.md](github/oz-erc721-erc1155.md) — Non-fungible tokens and multi-token patterns.
- **Gas Optimization:** [github/oz-gas-optimization.md](github/oz-gas-optimization.md) — Techniques to reduce gas costs.
- **Security Patterns:** [github/oz-security-patterns.md](github/oz-security-patterns.md) — Anti-patterns and recommended defenses.
- **Storage, Events, Errors:** [github/oz-storage-events-errors.md](github/oz-storage-events-errors.md) — Storage organization, event usage, and error handling.
- **Testing (Foundry/Hardhat):** [github/oz-testing-foundry-hardhat.md](github/oz-testing-foundry-hardhat.md) — Approaches and examples for testing.
- **Upgradeable Proxies:** [github/oz-upgradeable-proxies.md](github/oz-upgradeable-proxies.md) — Upgrade patterns and associated risks.

**How to use**

- **Read in suggested order:** start with the overview, then access control and security, followed by patterns (DeFi, tokens), and finally deployment and testing.
- **Quick references:** open any file from the "Skills (files)" section for details and examples.
